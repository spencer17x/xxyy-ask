# Feature Status

本文档记录 XXYY 客服应用当前已完成和计划中的用户、客服与运营可感知功能。

## User Features

用户可感知功能：

- [x] 产品功能问答：回答 XXYY 产品功能、配置步骤、Pro 权益、文档内容和官方 X 更新相关问题。
- [x] 来源引用展示：每条知识库回答都会返回引用来源，避免无依据编造产品细节。
- [x] 流式回答：Web UI 支持边生成边展示回答。
- [x] 视频附件展示：知识库片段里包含视频链接时，Web UI 可以展示视频附件。
- [x] 回答反馈：用户可以对回答提交正负反馈。
- [x] 边界问题回复：账户余额、订单、交易记录、代开通/代取消/代修改等业务动作执行请求、泛 MEV/链上取证、投资建议等问题目前会返回边界说明，不会假装查询实时数据或执行账户内操作。
- [x] 交易哈希识别：用户直接发送单笔完整交易哈希或交易链接时，系统可以进入夹子检测专用路径；用户说“帮我查这笔交易”或英文 `lookup this transaction` 并附上具体哈希时，也会按交易取证处理，不会被普通账户/交易记录查询边界抢走；同一哈希在文本和链接中重复出现时仍可识别，大小写差异不会误判为两笔；EVM 链只会从已支持的主网 explorer 域名推断，BaseScan、Base Blockscout、Etherscan、Ethereum Blockscout 和 BscScan 主网链接会分别识别为 Base、Base、Ethereum、Ethereum 和 BSC，`bsc.blockscout.com`、`optimistic.etherscan.io`、`sepolia.basescan.org`、`testnet.bscscan.com` 等未接入或测试网子域，以及 Polygon、Arbitrum、Avalanche、Linea、Scroll 等已知但未接入的 EVM explorer，不会被误判成 Ethereum、Base 或 BSC 主网；用户明确发送这些暂未支持的 explorer 链接，在文本里写 Base Sepolia、Ethereum Sepolia、BSC Testnet、ETH Goerli 等 EVM 测试网/开发网提示，或写 Polygon/MATIC/Polygon PoS/Polygon zkEVM、Arbitrum/ARB/Arbitrum One、Optimism/OP/Optimistic Ethereum、Avalanche/AVAX/Avalanche C-Chain、Linea、Scroll、Sonic、Berachain、Abstract、Gnosis/Gnosis Chain、Fantom/Fantom Opera、Moonriver、Mode/Mode Network、Taiko、World Chain、Zora/Zora Network、Manta/Manta Pacific、X Layer/XLayer/X-Layer、Plasma 等暂未支持链名时，会返回“其他链暂不支持”，并在复查信息里展示具体不支持的浏览器域名或链名，不会继续拿裸哈希探测已支持主网；如果用户文本里的链名和交易浏览器链接所属链冲突，例如 `Base` 搭配 Etherscan、`ETH` 搭配 Base Blockscout 或 Solscan 链接，会要求重新发送清晰的单笔交易引用，不会静默按其中一条链继续分析；裸 Solana 签名如果被用户标成 Base、Ethereum 或 BSC 等 EVM 链，或者 `0x` EVM 哈希被标成 Solana、`SOL 链` 或 `SOL chain`，也会要求重新确认链和哈希；Solana Explorer、Solscan 或 SolanaFM 链接如果在 query 或 URL fragment 里显式带 `cluster=devnet`、`cluster=testnet` 等非 mainnet 参数，也会返回暂不支持，不会按 mainnet Solana 继续分析；复制粘贴主网 Solana Explorer 链接时，`cluster=mainnet` 或 `cluster=mainnet-beta` 尾部多出的句号等标点会被忽略，避免误判为测试网；如果同一笔 EVM 哈希同时贴了多个不同链的 explorer 链接，系统不会抢判其中一条链；一次发送多笔不同交易哈希时，即使没有额外说明文字，也会要求用户拆成单笔，不会默认选择其中一笔；默认未接真实数据源时会明确提示暂未启用。
